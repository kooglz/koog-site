# 为 Koog-Site 添加内容管理后台 (CMS)

## 方案分析
您目前的网站是**静态网站**，托管在腾讯云 COS 上，通过 GitHub Actions 自动部署。这种架构非常先进且省钱。

关于您的需求：
1.  **之前的网站还能用吗？**
    *   **完全可以**。我们添加后台只是为了方便“修改 JSON 数据”，不会影响网站原本的展示逻辑。
2.  **可以在腾讯云上创建管理后台吗？**
    *   **可以，但不推荐**。如果在腾讯云买服务器（CVM）搭建后台（如 WordPress 或 Strapi），每月至少需要几十到上百元的服务器费用，且需要维护。
    *   **更好的方案**：使用 **Git-based CMS (Decap CMS)**。
        *   **原理**：后台就是一个网页，它直接修改您 GitHub 仓库里的 `works.json` 和图片。
        *   **费用**：**0 元**（完全免费）。
        *   **流程**：您在后台编辑 -> 点击保存 -> 自动推送到 GitHub -> 自动触发我们刚才修好的部署流程 -> 腾讯云更新。

## 实施计划

我将为您集成 **Decap CMS (原 Netlify CMS)**，这是一个专门为静态网站设计的后台系统。

### 1. 创建后台页面
*   在 `public/admin/` 目录下创建 `index.html`（后台入口）。
*   在 `public/admin/` 目录下创建 `config.yml`（后台配置文件）。

### 2. 配置数据结构 (Schema)
根据您现有的 `works.json`，我会配置以下编辑项：
*   **首页轮播图 (Hero Images)**：支持上传图片、修改 Alt 文本。
*   **作品集 (Works)**：支持增删改查作品，编辑标题、分类、年份、封面图、详细描述等。

### 3. 本地编辑模式
*   由于我们没有购买昂贵的认证服务器，我将为您配置**本地后台模式**。
*   **使用方法**：
    1.  在终端运行一个简单的命令启动后台代理。
    2.  打开 `http://localhost:3000/admin` 即可进入漂亮的后台界面。
    3.  编辑完保存，它会自动修改本地的 `json` 文件。
    4.  您像往常一样 `git push`，网站就更新了。

## 预期效果
您将拥有一个可视化的管理后台，可以像编辑 Word 一样编辑网站内容，无需再手动修改代码或 JSON 文件。
